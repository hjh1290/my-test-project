사이버 로그북

원격 저장소 연결부터 인증 문제 해결까지, 오늘의 GitHub 전투 기록을 네온 시티 감성으로 정리한 페이지입니다.

STEP // 01

원격 저장소 연결

로컬 프로젝트를 GitHub 원격 저장소와 연결하고 저장소 URL 정확성을 확인했습니다.

STEP // 02

인증 문제 돌파

권한 오류를 추적해 인증 방식을 바로잡고 접근 흐름을 안정화했습니다.

STEP // 03

최종 푸시 완료

브랜치 기록을 정리한 뒤 첫 푸시를 완료하고 원격 동기화를 확인했습니다.

에러 추적 로그

  1. repository not found - URL 오타를 수정하고 저장소 권한을 다시 확인
  2. permission denied - 인증 설정을 점검하고 접근 방식을 전환
  3. non-fast-forward - 원격 변경사항 동기화 후 충돌 해결, 재푸시
신호 수신: "에러는 적이 아니라 디버깅 힌트다."
